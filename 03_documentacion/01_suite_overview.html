Sobre la suite

¿Qué es y por qué existe?

IBM Integration Designer exporta aplicaciones completas en archivos .twx (archivos ZIP con XML propietario). Sin herramientas de análisis, hacer ingeniería inversa sobre ellos implica abrir XMLs a mano — un proceso lento y propenso a errores.

🗜️
Formato .TWX
Un archivo .twx es un ZIP que contiene decenas de XMLs con namespaces IBM propietarios. La suite los abre, clasifica y parsea automáticamente.
.bpd .service .bo .decision .htm .integr
🔍
Ingeniería inversa completa
Extrae procesos, servicios, Business Objects, scripts JavaScript embebidos, endpoints REST/SOAP, tablas de decisión y genera documentación Markdown con diagramas Mermaid en segundos.
🤖
GitHub Copilot nativo
La extensión VS Code registra el participante @nttdata en Copilot Chat. Basta escribir @nttdata v1.0 flows archivo.twx para obtener diagramas Mermaid interactivos.
📐
Sin dependencias externas
Solo Python stdlib (zipfile, xml.etree). No requiere instalar IBM Designer, Docker ni librerías de terceros para analizar archivos TWX.

Módulos del backend Python

twx_parser.py
Abre el ZIP, detecta artefactos por extensión, parsea XML con namespaces IBM BPM y retorna un TWXPackage completo.
entity_extractor.py
Extrae Business Objects, sus campos tipados, herencias, namespaces y variables declaradas en procesos/servicios.
service_extractor.py
Lee cada paso de servicio: scripts JS, llamadas a servicios externos, SQL embebido, bindings REST/SOAP y mapeos de parámetros.
flow_mapper.py
Reconstruye el grafo de flujo de cada proceso (nodos, transiciones, gateways) y genera el grafo de dependencias entre artefactos.
doc_generator.py
Orquesta todos los extractores y genera documentación Markdown completa con tablas, diagramas Mermaid y secciones navegables.
banner.py
Banner ASCII del logo NTT DATA (símbolo Shonai + letras de bloque) con colores ANSI azul corporativo para terminal y Copilot Chat.
CLI Reference

Comandos disponibles

Todos los comandos siguen el mismo patrón: python -m ibm_twx_tools <cmd> archivo.twx

analyze
Análisis general Resumen JSON del paquete: nombre, versión, toolkit flag y conteo de artefactos por tipo.
entities
Business Objects Lista todos los BOs con sus campos, tipos, flags de lista/requerido y relaciones de herencia.
services
Servicios e implementación Extrae cada servicio con sus pasos, inputs/outputs, tipo de paso y dependencias.
flows
Flujos de proceso (Mermaid) Genera un diagrama Mermaid flowchart TD por cada proceso/servicio del paquete.
deps
Grafo de dependencias Diagrama Mermaid graph LR con todas las llamadas entre artefactos del TWX.
docs
Documentación completa Markdown listo para publicar: resumen, entidades, servicios, scripts, endpoints y diagramas.
scripts
Scripts embebidos Vuelca todo el código JavaScript/Groovy encontrado en los pasos de servicio.
endpoints
Endpoints externos Lista todas las URLs REST/SOAP referenciadas en los servicios de integración.
entries
Entry Points Identifica los artefactos que no son invocados por ningún otro — puntos de entrada de la app.

Proceso interno

¿Cómo funciona internamente?

Pipeline de análisis en 5 fases, desde el archivo .twx hasta el output final.

1

Apertura del TWX

TWXParser abre el archivo como ZIP, lee el MANIFEST.MF para obtener nombre, GUID y versión de la app, luego enumera todos los archivos XML internos clasificándolos por extensión.

2

Parsing XML con namespaces IBM

Cada artefacto se parsea con xml.etree.ElementTree mapeando los namespaces propietarios de IBM BPM / Lombardi 8.0 y BAW 20.0. Se extraen GUIDs, versiones y descripciones.

3

Extracción especializada

Cada extractor lee el árbol XML del artefacto correspondiente: EntityExtractor busca <field>, <attribute>; ServiceExtractor busca <scriptTask>, <sequenceFlow>, etc.

4

Construcción del grafo

FlowMapper recorre todos los artefactos, resuelve referencias cruzadas (<calledService>, <calledElement>) y construye el grafo de dependencias inter-artefacto.

5

Generación de output

DocGenerator orquesta los 4 extractores anteriores y produce Markdown con tablas, bloques de código y diagramas flowchart TD renderizables directamente en GitHub o VS Code.
