Lo más importante que entendimos

¿Qué es un archivo .TWX y por qué es difícil?

Estructura interna del TWX

Un .twx es simplemente un ZIP. Adentro tiene una carpeta objects/ con cientos de archivos XML nombrados así:

objects/
  1.a3f2c1d4.xml    ← un servicio
  12.b7e9f2a1.xml   ← un Business Object
  25.c4d8e3b2.xml   ← un BPD (proceso)
  64.f1a2b3c4.xml   ← una pantalla Coach View
  62.e5f6a7b8.xml   ← variables de entorno

El número antes del punto es el tipo de artefacto. Eso es todo lo que tienes para orientarte — no hay carpetas, no hay índice.

El corazón técnico

Los 9 comandos que ejecuta run_analysis.py

Cuando corres python run_analysis.py Profuturo_Redencion.twx, el script ejecuta estos 9 pasos en secuencia de forma automática:

# 1. Extrae y clasifica todos los artefactos del TWX
python twx_parser.py archivo.twx              → 232 artefactos clasificados

# 2. Extrae entidades (Business Objects)
python entity_extractor.py --twx archivo.twx  → entities.json

# 3. Extrae todos los servicios (IS + GSS + HHS)
python service_extractor.py --twx archivo.twx → services.json

# 4. Extrae y clasifica flujos BPD
python flow_mapper.py --twx archivo.twx       → flows.json

# 5. Extrae scripts JavaScript de cada paso
python script_extractor.py --twx archivo.twx  → scripts/ (455 KB)

# 6. Extrae variables de entorno (62.*.xml)
python env_extractor.py --twx archivo.twx     → env_vars.json

# 7. Extrae UCAs y correlaciones
python uca_extractor.py --twx archivo.twx     → ucas.json

# 8. Extrae pantallas (Coach Views 64.*.xml)
python coach_extractor.py --twx archivo.twx   → coaches.json

# 9. Genera el prompt maestro para Copilot
python prompt_builder.py                      → COPILOT_PROMPT.md (57 KB)
Lo que aprendimos en el camino

Las 11 correcciones críticas — Metodología v3

Después de analizar Profuturo con v1 y v2, el equipo detectó errores en los resultados. Estas son las 11 correcciones que aplicamos para llegar a v3:

1. Orden de pasos — sequenceFlow, no coordenadas x/y

En v1 usábamos las coordenadas visuales de los nodos para inferir el orden. Error: el diseñador puede mover nodos libremente. El orden real está en las etiquetas <sequenceFlow> del XML.

2. Botones extraídos de Coach Views (64.*.xml), no de descripciones

En v1 leíamos el texto de descripción del HHS para inferir botones. Error: las descripciones son libres y a veces están vacías. Los botones reales están en los XMLs de 64.* como boundary events.

3. PostponeAction = "Cerrar tarea" (no "Posponer")

El botón PostponeAction en IBM BPM se llama "Salir" o "Posponer Tarea" en la UI, pero su comportamiento real es cerrar la tarea del usuario. El nombre engaña.

4. IS "IG Revisión UDI" es SÍNCRONO

En v1 fue clasificado como asíncrono (igual que los otros IS) porque también tiene un GSS de respuesta. Error: el servicio IIB se invoca de forma síncrona. El GSS existe para compatibilidad pero no hay espera UCA.

5. Variables de entorno en 62.*.xml, NO en el MANIFEST

El archivo META-INF/MANIFEST.MF solo contiene Manifest-Version: 1.0. Todas las constantes y variables de entorno están en los XMLs con prefijo 62.*.

6. ID_SUBETAPA_GEN_ARCH_RESP: 8627 (Dev/Test/Stage) vs 8628 (Producción)

Este es el único ID que difiere entre ambientes. Todos los demás son iguales en los 4 ambientes. Este diferenciador es crítico para evitar errores en producción.

7–11. Correcciones de estructura XML interna

Nombre del proceso desde el primer BPD encontrado · Tipo de servicio por campo processType (3=HHS, 4=IS, 6=GSS) · Los BOs están en 12.* no en carpetas nombradas · Los UCAs en 4.* tienen el folio como clave de correlación · Los scripts JS están embebidos en nodos tipo scriptTask.

Lo que el equipo Profuturo corrigió

Después de revisar v1 y v2 del análisis, el equipo señaló estas correcciones que incorporamos en v3:

IG Revisión UDI → Síncrono

Lo habíamos clasificado como asíncrono. El equipo confirmó que es síncrono.

Botones de XML real, no de descripciones

Cambiamos la fuente de extracción de texto libre a 64.*.xml.

Orden de pasos por sequenceFlow

Eliminamos el orden por coordenadas visuales (x/y) que usábamos antes.

Recomendación de migración 1:1 eliminada

La v2 incluía sugerencias de migración que el equipo consideró prematuras para esta etapa.
