El skill principal

twx-suite — Skill Maestro

twx-suite

Skill orquestador. Cuando dices "analiza este TWX" o "reverse engineer este proceso", Copilot usa este skill para decidir qué ciclos ejecutar y en qué orden. Conoce los 9 comandos y cuándo usar cada uno.

Los 9 ciclos de extracción

Skills individuales — uno por ciclo

🔍
twx-analyze
Ciclo 1 — Inventario
Genera el resumen completo del TWX: nombre de la app, versión, si es toolkit o aplicación, y conteo de artefactos por tipo. Siempre es el primer ciclo a ejecutar.
python -m ibm_twx_tools analyze "archivo.twx"
Produce: JSON con app_name, version, total_artifacts, by_type
JSONVista generalPrimer paso
📦
twx-entities
Ciclo 2 — Modelo de datos
Extrae todos los Business Objects con sus campos, tipos de datos, herencia y relaciones. Es el modelo de datos del sistema IBM BPM.
python -m ibm_twx_tools entities "archivo.twx"
Produce: JSON array de BOs con campos tipados
JSONBusiness ObjectsTipos de datos
⚙️
twx-services
Ciclo 3 — Servicios
Extrae IS, GSS y HHS con sus pasos en orden sequenceFlow, inputs/outputs y dependencias. Distingue tipos por processType (3=HHS, 4=IS, 6=GSS).
python -m ibm_twx_tools services "archivo.twx"
Produce: JSON con servicios, pasos, IO, dependencias
IS / GSS / HHSsequenceFlow
🔀
twx-flows
Ciclo 4 — Diagramas
Genera diagramas Mermaid flowchart TD por cada proceso y servicio. Nodos diferenciados: inicio, fin, gateways, scripts, servicios externos.
python -m ibm_twx_tools flows "archivo.twx"
Produce: Bloques Mermaid renderizables en Markdown/HTML
MermaidVisualizaciónBPDs
🕸️
twx-deps
Ciclo 5 — Dependencias
Construye el grafo de dependencias entre artefactos (quién llama a quién). Identifica artefactos críticos, huérfanos y dependencias circulares.
python -m ibm_twx_tools deps "archivo.twx"
Produce: Mermaid graph LR con relaciones caller→callee
Grafo Impacto
📄
twx-docs
Ciclo 6 — Documentación
Genera la documentación técnica completa en Markdown: resumen, BOs, servicios, scripts, endpoints, diagramas y entrypoints. Todo en un solo archivo.
python -m ibm_twx_tools docs "archivo.twx" -o "docs.md"
Produce: Markdown completo listo para compartir
MarkdownEntregable
</>
twx-scripts
Ciclo 7 — Código JS
Extrae y etiqueta todo el JavaScript embebido en los pasos de los servicios. Muestra el artefacto, el paso y el código. En Profuturo generó 455 KB de scripts.
python -m ibm_twx_tools scripts "archivo.twx"
Produce: Bloques JS por artefacto y paso (etiquetados)
JavaScripttw.local.*455 KB
🌐
twx-endpoints
Ciclo 8 — Integraciones externas
Lista todas las URLs REST, SOAP/WSDL y HTTP referenciadas. Detecta URLs hardcodeadas, diferencias Dev/Prod y conexiones críticas para migración.
python -m ibm_twx_tools endpoints "archivo.twx"
Produce: JSON con tipo, método y URL por servicio
RESTSOAP/WSDLURLs
🚪
twx-entries
Ciclo 9 — API pública
Identifica los puntos de entrada: artefactos que no son llamados por ningún otro. Son la API pública y los puntos de inicio del sistema IBM BPM.
python -m ibm_twx_tools entries "archivo.twx"
Produce: JSON de artefactos sin callers (puntos de entrada)
API públicaEntry points
Skill especializado

profuturo-twx — Metodología v3

Skill de proyecto

profuturo-twx

Skill específico para proyectos Profuturo. Contiene la taxonomía de prefijos IBM BPM de Profuturo, las 11 correcciones v3 de la metodología y los patrones de extracción validados con el equipo.

Orden por sequenceFlowNunca por coordenadas X/Y del diseñador visual
Botones de 64.*.xmlLos Coach Views son la fuente real — no las descripciones
PostponeAction = Cerrar tareaNo "Posponer" ni "Cancelar" — es cerrar la tarea del usuario
IG Revisión UDI = SÍNCRONOA pesar de tener GSS de respuesta, el IS es síncrono
Vars entorno en 62.*.xmlEl MANIFEST.MF solo dice Manifest-Version: 1.0 — no sirve
ID 8627 Dev ≠ 8628 ProdEl único ID que difiere entre ambientes — crítico en producción

En la práctica

¿Cómo se ejecuta todo esto?

Opción A — Ciclo completo automatizado (run_analysis.py)

Un solo comando ejecuta los 9 ciclos en secuencia y genera el COPILOT_PROMPT.md listo para Copilot:

# Desde la raíz del proyecto
python run_analysis.py "C:\ruta\al\archivo.twx"

# Resultado en ~10 minutos:
✓  Ciclo 1 — analyze    →  232 artefactos clasificados
✓  Ciclo 2 — entities   →  54 Business Objects extraídos
✓  Ciclo 3 — services   →  74 servicios con pasos
✓  Ciclo 4 — flows      →  44 diagramas Mermaid
✓  Ciclo 5 — deps       →  grafo de dependencias
✓  Ciclo 6 — docs       →  documentación Markdown (378 KB)
✓  Ciclo 7 — scripts    →  JavaScript extraído (455 KB)
✓  Ciclo 8 — endpoints  →  12 endpoints SOAP
✓  Ciclo 9 — entries    →  puntos de entrada identificados
✓  COPILOT_PROMPT.md generado  →  57 KB listo para Copilot CLI

Opción B — Ciclo individual desde Copilot CLI

Puedes pedir un ciclo específico directamente en el chat:

# Le dices a Copilot:
"Extrae los endpoints de este TWX: C:\ruta\archivo.twx"

# Copilot activa el skill twx-endpoints y ejecuta:
python -m ibm_twx_tools endpoints "archivo.twx"

# Luego interpreta el JSON y te lo presenta en tabla
