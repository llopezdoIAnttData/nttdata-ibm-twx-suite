⚠️ Los artefactos de complejidad 🟢 Baja se generan automáticamente 1:1. Los de 🔴 Alta requieren intervención del LLM para "reimaginar" en el paradigma Appian — no es traducción directa, es reingeniería asistida por IA.

Plan de ejecución
5 fases para llegar a la automatización completa
F1
INGENIERÍA INVERSA
1 sesión
Entender el formato Appian
Analizar el .zip de exportación de Appian para descubrir el schema XML exacto de cada tipo de objeto. Sin esto no podemos generar nada compatible.
Descomprimir el .zip de exportación de Appian
Catalogar la estructura de carpetas y archivos
Extraer el schema XML de Record Type, Process Model, Interface, Integration, Constant
Leer documentación en docs.appian.com para campos obligatorios
Identificar la versión de Appian del entorno Profuturo
📦 Entregable: mapping_schema.json — schemas XML de cada objeto Appian
F2
TABLAS DE MAPEO
1-2 sesiones
Reglas de transformación IBM → Appian
Con el schema de Appian conocido, definir las reglas exactas de cómo cada campo IBM se convierte en su equivalente Appian. Los casos complejos (JS → Expression Language) los resuelve el LLM.
Mapeo completo BO → Record Type (campo a campo)
Mapeo Variables de entorno → Constants
Mapeo Endpoints → Integration Objects / Connected Systems
Reglas LLM para traducción de scripts JavaScript → Appian Expression Language
Reglas LLM para reimplementación de flujos BPD → Process Model Appian
📦 Entregable: mapping_rules.json — reglas de transformación completas
F3
CONSTRUCCIÓN
3-5 sesiones
Módulo appian_generator
Construir los builders Python para cada tipo de objeto, empezando por los de baja complejidad (BOs, Constants) y terminando con los de alta complejidad (Process Models, Interfaces).
Sprint 1: record_type_builder.py + constant_builder.py (BOs y variables)
Sprint 2: integration_builder.py (endpoints REST/SOAP)
Sprint 3: expression_rule_builder.py (scripts → Expression Language con LLM)
Sprint 4: process_model_builder.py (BPDs → Process Models)
Sprint 5: interface_builder.py + package_assembler.py
📦 Entregable: módulo appian_generator funcional
F4
INTEGRACIÓN CLI
1 sesión
Skill /twx-to-appian en Copilot CLI
Integrar el generador como un nuevo skill del suite NTT DATA. Un solo comando hace el pipeline completo: .twx → análisis → .zip Appian.
Crear SKILL.md para twx-to-appian con instrucciones agentic
Integrar en el menú /nttdata (opción 15)
Agregar comando al CLI de ibm_twx_tools: python -m ibm_twx_tools to-appian
Pushear al repositorio GitHub
📦 Entregable: skill /twx-to-appian publicado en GitHub
F5
VALIDACIÓN
1-2 sesiones
Import real en consola Appian — Redención de Bono
Prueba end-to-end: generar el paquete desde el TWX de Redención de Bono e importarlo en la consola Appian real. Iterar hasta importación limpia.
Correr /twx-to-appian sobre Profuturo_RedencionBono.twx
Import del .zip generado en consola Appian
Revisar errores de validación y corregir schemas
Validar Record Types, Constants e Integrations en Appian Designer
Documentar ajustes necesarios para los demás procesos
📦 Entregable: primer proceso Profuturo importado automáticamente en Appian ✅
